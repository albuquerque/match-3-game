## üìù Documentation Standards

### File Naming
- Use UPPERCASE for main docs (e.g., `FEATURES.md`)
- Use descriptive names
- End with `.md` extension

### Content Structure
- Start with a clear title (# Heading)
- Include a brief overview
- Use sections (## Headings) to organize
- Code examples in ```language blocks
- Keep up to date with latest implementation

### Updating Documentation
When you add a new feature:
1. Update [FEATURES.md](FEATURES.md) with the new feature
2. Create a system-specific doc if needed (like LEVELS_README.md)
3. Update this README.md index
